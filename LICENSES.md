# License and External Asset Policy

## Principles

1. Do not redistribute datasets, checkpoints or code merely because they are downloadable.
2. Repository code license, pretrained weight terms, paper license and dataset license may differ.
3. Record the exact terms accepted, date, user/institution and source page.
4. `unknown` means use is blocked until reviewed.
5. This file is operational guidance, not legal advice.

## Verified examples at snapshot time

- CompressAI repository contains a BSD-style 3-clause license with an explicit statement that no patent license is granted.
- MLIC repository contains Apache-2.0.
- PARSeq is primarily Apache-2.0; reproduced ABINet and CRNN components carry their own BSD/MIT notices.
- JPEG AI reference software page identifies BSD 3-Clause; standard documents and patent/licensing considerations remain separate.
- RDVQ repository advertises CC-BY-4.0 at the inspected snapshot; verify whether that applies to all code, weights and third-party components before integration.

The machine-readable source and review state live in `literature/repositories.yaml` and `data/registry.yaml`.
