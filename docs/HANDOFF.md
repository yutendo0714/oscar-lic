## Key Diagnosis

The current bottleneck is code assignment ranking, not payload accounting or gate recall.

- Top-8 assignment oracle headroom is now actual-bitstream verified at unchanged selected-candidate and code-index payload size: `-24` unicode / `-19` latin Tesseract edit characters with PARSeq unchanged.
- The promoted image-diff gate recovers only 4 oracle-change groups. All 13 missed oracle-change groups are scorer-ranking failures; the gate is not rejecting oracle-best choices.
- Relaxing the current gate is unsafe: blindly taking the current scorer best on missed groups sums to `+2` Tesseract edits.
- Row AUC/AP is frequently misleading. Multiple high-AUC variants worsen policy-level OCR or introduce many false changes.
